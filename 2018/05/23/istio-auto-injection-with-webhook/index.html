еүҚиЁҖ

Kubernets 1.9зүҲжң¬еј•е…ҘдәҶAdmission Webhook(web еӣһи°ғ)жү©еұ•жңәеҲ¶пјҢйҖҡиҝҮWebhook,ејҖеҸ‘иҖ…еҸҜд»ҘйқһеёёзҒөжҙ»ең°еҜ№Kubernets API Serverзҡ„еҠҹиғҪиҝӣиЎҢжү©еұ•пјҢеңЁAPI ServerеҲӣе»әиө„жәҗж—¶еҜ№иө„жәҗиҝӣиЎҢйӘҢиҜҒжҲ–иҖ…дҝ®ж”№гҖӮ

дҪҝз”Ёwebhookзҡ„дјҳеҠҝжҳҜдёҚйңҖиҰҒеҜ№API Serverзҡ„жәҗз ҒиҝӣиЎҢдҝ®ж”№е’ҢйҮҚж–°зј–иҜ‘е°ұеҸҜд»Ҙжү©еұ•е…¶еҠҹиғҪгҖӮжҸ’е…Ҙзҡ„йҖ»иҫ‘е®һзҺ°дёәдёҖдёӘзӢ¬з«Ӣзҡ„webиҝӣзЁӢпјҢйҖҡиҝҮеҸӮж•°ж–№ејҸдј е…ҘеҲ°kubernetsдёӯпјҢз”ұkubernetsеңЁиҝӣиЎҢиҮӘиә«йҖ»иҫ‘еӨ„зҗҶж—¶еҜ№жү©еұ•йҖ»иҫ‘иҝӣиЎҢеӣһи°ғгҖӮ

Istio 0.7зүҲжң¬е°ұеҲ©з”ЁдәҶKubernets webhookе®һзҺ°дәҶsidecarзҡ„иҮӘеҠЁжіЁе…ҘгҖӮ

д»Җд№ҲжҳҜAdmission

AdmissionжҳҜKubernetsдёӯзҡ„дёҖдёӘжңҜиҜӯпјҢжҢҮзҡ„жҳҜKubernets API Serverиө„жәҗиҜ·жұӮиҝҮзЁӢдёӯзҡ„дёҖдёӘйҳ¶ж®өгҖӮеҰӮдёӢеӣҫжүҖзӨәпјҢеңЁAPI ServerжҺҘж”¶еҲ°иө„жәҗеҲӣе»әиҜ·жұӮж—¶пјҢйҰ–е…ҲдјҡеҜ№иҜ·жұӮиҝӣиЎҢи®ӨиҜҒе’ҢйүҙжқғпјҢз„¶еҗҺз»ҸиҝҮAdmissionеӨ„зҗҶпјҢжңҖеҗҺеҶҚдҝқеӯҳеҲ°etcdгҖӮ д»ҺеӣҫдёӯзңӢеҲ°пјҢAdmissionдёӯжңүдёӨдёӘйҮҚиҰҒзҡ„йҳ¶ж®өпјҢMutationе’ҢValidationпјҢиҝҷдёӨдёӘйҳ¶ж®өдёӯжү§иЎҢзҡ„йҖ»иҫ‘еҰӮдёӢпјҡ * Mutation

MutationжҳҜиӢұж–ҮвҖңзӘҒеҸҳвҖқзҡ„ж„ҸжҖқ,д»Һеӯ—йқўдёҠеҸҜд»ҘзҹҘйҒ“еңЁMutationйҳ¶ж®өеҸҜд»ҘеҜ№иҜ·жұӮеҶ…е®№иҝӣиЎҢдҝ®ж”№гҖӮ * Validation

еңЁValidationйҳ¶ж®өдёҚе…Ғи®ёдҝ®ж”№иҜ·жұӮеҶ…е®№пјҢдҪҶеҸҜд»Ҙж №жҚ®иҜ·жұӮзҡ„еҶ…е®№еҲӨж–ӯжҳҜз»§з»ӯжү§иЎҢиҜҘиҜ·жұӮиҝҳжҳҜжӢ’з»қиҜҘиҜ·жұӮгҖӮ

Admission webhook

йҖҡиҝҮAdmission webhook,еҸҜд»ҘеҠ е…ҘMutationе’ҢValidationдёӨз§Қзұ»еһӢзҡ„webhookжҸ’д»¶пјҢиҝҷдәӣжҸ’д»¶е’ҢKubernetsжҸҗдҫӣзҡ„йў„зј–иҜ‘зҡ„AdmissionжҸ’д»¶е…·жңүзӣёеҗҢзҡ„иғҪеҠӣгҖӮеҸҜд»ҘжғіеҲ°зҡ„з”ЁйҖ”еҢ…жӢ¬пјҡ * дҝ®ж”№иө„жәҗгҖӮдҫӢеҰӮIstioе°ұйҖҡиҝҮAdmin WebhookеңЁPodиө„жәҗдёӯеўһеҠ дәҶEnvoy sidecarе®№еҷЁгҖӮ * иҮӘе®ҡд№үж ЎйӘҢйҖ»иҫ‘пјҢдҫӢеҰӮеҜ№иө„жәҗеҗҚз§°жңүдёҖдәӣзү№ж®ҠиҰҒжұӮгҖӮжҲ–иҖ…еҜ№иҮӘе®ҡд№үиө„жәҗзҡ„еҗҲжі•жҖ§иҝӣиЎҢж ЎйӘҢгҖӮ

йҮҮз”ЁWebhookиҮӘеҠЁжіЁе…ҘIstio Sidecar

KubernetsзүҲжң¬иҰҒжұӮ

webhookж”ҜжҢҒйңҖиҰҒKubernets1.9жҲ–иҖ…жӣҙй«ҳзҡ„зүҲжң¬,дҪҝз”ЁдёӢйқўзҡ„е‘Ҫд»ӨзЎ®и®Өkube-apiserverзҡ„Admin webhookеҠҹиғҪе·ІеҗҜз”ЁгҖӮ

kubectl api-versions | grep admissionregistration

admissionregistration.k8s.io/v1beta1

з”ҹжҲҗsidecar injection webhookзҡ„еҜҶй’Ҙе’ҢиҜҒд№Ұ

WebhookдҪҝз”Ёж•°еӯ—иҜҒд№Ұеҗ‘kube-apiserverиҝӣиЎҢиә«д»Ҫи®ӨиҜҒпјҢеӣ жӯӨйңҖиҰҒе…ҲдҪҝз”Ёе·Ҙе…·з”ҹжҲҗеҜҶй’ҘеҜ№пјҢе№¶еҗ‘Istio CAз”іиҜ·ж•°еӯ—иҜҒд№ҰгҖӮ

./install/kubernetes/webhook-create-signed-cert.sh /
    --service istio-sidecar-injector /
    --namespace istio-system /
    --secret sidecar-injector-certs

е°Ҷз”ҹжҲҗзҡ„ж•°еӯ—иҜҒд№Ұй…ҚзҪ®еҲ°webhookдёӯ

cat install/kubernetes/istio-sidecar-injector.yaml | /
     ./install/kubernetes/webhook-patch-ca-bundle.sh > /
     install/kubernetes/istio-sidecar-injector-with-ca-bundle.yaml

еҲӣе»әsidecar injection configmap

kubectl apply -f install/kubernetes/istio-sidecar-injector-configmap-release.yaml

йғЁзҪІsidecar injection webhook

kubectl apply -f install/kubernetes/istio-sidecar-injector-with-ca-bundle.yaml

йҖҡиҝҮе‘Ҫд»ӨжҹҘзңӢйғЁзҪІеҘҪзҡ„webhook injector

```` kubectl -n istio-system get deployment -listio=sidecar-injector Copy NAME DESIRED CURRENT UP-TO-DATE AVAILABLE AGE istio-sidecar-injector 1 1 1 1 1d


### ејҖеҗҜйңҖиҰҒиҮӘеҠЁжіЁе…Ҙsidecarзҡ„namespace

kubectl get namespace -L istio-injection

```
